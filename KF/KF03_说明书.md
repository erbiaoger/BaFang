# KF03 手动标注车辆轨迹说明书

本说明书适用于 `KF03.py`，用于从多通道传感器数据中手动标注车辆轨迹，并通过卡尔曼滤波辅助补全。

## 功能概述

- 读取指定目录下的 `.sac` 数据，生成水平剖面图（section）用于可视化。
- 鼠标点击快速标注车辆轨迹起点，自动沿空间方向跟踪。
- 右键两点补全轨迹区间，支持邻域峰值吸附。
- 键盘快捷键保存、删除、切换数据窗口与数据版本。

## 运行环境与依赖

- Python 3.9+
- 依赖库：`numpy`、`scipy`、`matplotlib`、`obspy`、`PyQt6`、`pandas`

安装示例（按需）：

```bash
pip install numpy scipy matplotlib obspy PyQt6 pandas
```

## 数据准备

脚本默认读取路径（可按需修改）：

- `self.files`：原始数据目录，示例：`/Volumes/SanDisk2T/BaFang/Track_data/Origin/2025-05-19/10/`
- 文件格式：`*.sac`

注意事项：

- 目录结构与命名需保持一致，否则切换数据窗口/数据版本时会找不到文件。
- 采样间隔默认 `dt = 0.001` 秒，窗口大小为 `1000 * 60 * 15`（15 分钟）。

## 运行方式

```bash
python KF03.py
```

启动后会打开可交互的图形界面。

## 操作说明（鼠标）

- **左键**：在当前点击位置附近寻找峰值，作为起点进行卡尔曼跟踪，生成一条候选轨迹。
- **中键**：清空当前未保存的候选轨迹。
- **右键**：两次右键确定起止点，在区间内线性插值并吸附峰值补全轨迹。

## 操作说明（键盘）

- **S**：保存当前候选轨迹到 `veh_states`，并清空当前候选轨迹。
- **Z**：对当前候选轨迹进行多项式拟合补全（包含前后端）。
- **D**：保存全部轨迹到 `npz` 文件，并跳转到下一个时间窗口。
- **Backspace**：删除最后一条已保存轨迹。
- **A**：切换到 AGC 数据版本（从 `ltasta` 目录切换为 `agc`）。
- **Q**：从 AGC 恢复回原始版本。
- **F**：切换到下一小时目录（如 `10 -> 11`），并重置窗口。

## 输出文件

- 轨迹保存路径：当前工作目录
- 保存格式：`npz`
- 命名规则：`peaks_YYYYMMDDHH_XX.npz`

说明：

- `XX` 为窗口序号，从 00 开始递增。
- 文件内键为 `veh_states`。

## 参数说明（核心）

在 `update_tracking_data()` 中可调整：

- 检测参数 `detect`：
  - `prominence`：峰值显著性
  - `distance`：峰间最小间隔（采样点）
  - `wlen`：峰值宽度窗口
  - `height`：峰值高度阈值
- 车辆参数 `veh`：
  - `vel_init`：初始速度
  - `vmin/vmax`：速度约束范围
  - `tmin/tmax`：峰值搜索窗口（秒）
  - `dt/dx`：采样间隔、空间步长

## 常见问题

- **点击无峰值**：提示“无检测到峰值”，需提高 `prominence` 或调整 `height`。
- **轨迹断裂**：尝试右键补全区间，或使用 `Z` 拟合补全。
- **数据窗口越界**：会提示“已经到达数据末尾”。

## 修改建议

- 路径与窗口大小可根据数据源调整。
- 若峰值过密导致误匹配，可增大 `distance` 或缩小 `tmin/tmax`。
- 若速度变化较大，可放宽 `vmin/vmax`。
